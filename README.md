# Ideal Company Platform v8

نسخة **خادم مركزي + واجهة React** من منصة **الشركة المثالية**.

في هذه المرحلة أصبحت المنصة تعمل بطريقتين:
- **تشغيل مباشر** من النسخة المبنية عبر الخادم المحلي المركزي.
- **تطوير** عبر Vite للواجهة، مع API خلفي وSQLite مركزي.

## ما الذي تغيّر في هذه النسخة؟

- نقل البيانات من `localStorage` فقط إلى **خادم Node.js** مع **SQLite**.
- فصل **بيانات المنصة المشتركة** عن **جلسة كل مستخدم**.
- حفظ مركزي للمدارس والطلاب والمستخدمين والسجلات.
- جلسات دخول حقيقية عبر **رمز جلسة Session Token**.
- استمرار صلاحيات:
  - الأدمن العام
  - مدير المدرسة
  - المعلم
  - المشرف
  - البوابة
  - الطالب
- استمرار:
  - QR لكل طالب
  - بصمة الوجه بالصورة أو بالكاميرا
  - إجراءات المعلم: مكافأة / خصم / برنامج
  - استيراد الطلاب من ملف نور Excel
- دمج حماية أفضل في الخادم عند الحفظ:
  - الأدمن العام يحفظ المنصة كاملة
  - المستخدمون المدرسيون يحفظون نطاق مدرستهم فقط

## التشغيل السريع

هذه النسخة تحتوي على **واجهة مبنية مسبقًا** داخل `dist`، لذلك يكفي:

```bash
npm start
```

ثم افتح:

```bash
http://localhost:4000
```

> لا تحتاج `npm install` لتشغيل النسخة الجاهزة ما دام مجلد `dist` موجودًا.

## تشغيل التطوير

إذا أردت التعديل على الواجهة وإعادة البناء:

```bash
npm install
npm run server
npm run dev
```

ثم افتح الواجهة عادة على:

```bash
http://localhost:5173
```

ويعمل الـ API الخلفي على:

```bash
http://localhost:4000
```

## إعادة البناء

```bash
npm install
npm run build
```

## الملفات المهمة

- `server/index.js` — الخادم الخلفي
- `server/state.js` — تهيئة البيانات الافتراضية وهياكل الصلاحيات
- `data/platform.db` — قاعدة البيانات SQLite (تنشأ تلقائيًا)
- `src/App.jsx` — الواجهة الرئيسية

## ملاحظات معمارية

هذه النسخة أصبحت **أقرب لمنصة فعلية** وليست مجرد واجهة، لكن ما زال أمامها تطوير إنتاجي لاحق، مثل:

- تشفير كلمات المرور بدل حفظها بصيغتها الحالية
- رفع صور الوجه كملفات منظمة بدل الاعتماد على Data URL
- فصل إعدادات كل مدرسة عن الإعدادات العامة بشكل كامل
- واجهات API تفصيلية بدل حفظ لقطة حالة كبيرة
- تدقيق أعمق للصلاحيات على مستوى كل عملية
- دعم نشر سحابي ومزامنة أجهزة المدارس بشكل إنتاجي

## ملاحظة مهمة

يعتمد الخادم على `node:sqlite` المدمج في Node.js 22، لذلك يفضّل تشغيله على **Node 22 أو أحدث**.

## تنبيه مهم إذا ظهرت رسالة دخول خاطئة

إذا فتحت الواجهة على `5173` فيلزم تشغيل الخادم الخلفي أيضاً بأمر:

```bash
npm run server
```

أما إذا أردت تشغيل النسخة الجاهزة كاملة فافتحها من:

```bash
http://localhost:4000
```

وفي هذا الإصدار تمت إضافة **ترحيل تلقائي** لقاعدة البيانات القديمة، لذلك إذا كان لديك ملف `data/platform.db` من نسخة سابقة فسيتم إصلاح جداول الجلسات تلقائياً عند التشغيل.
